touch： 创建文件 如：touch index.html

###Git工作原理 

对于任何一个文件，在git内都有三种状态  
    1.已提交（committed）:表示该文件已经被安全地保存在本地数据库中  
    2.已修改（modified）：表示修改了某个文件，但还没有提交保存  
    3.已暂存（staged）：表示把已经修改的文件放在下次提交时要保存的清单中

Git管理项目时，文件流分为三个工作区域：
1.Git 的工作目录
2.暂存区域
3.本地仓库

git命令

###配置用户

git config –global user.name “your name”
git config –global user.email “your email”

###初始化仓库
git init

###查看文件状态
git status

###添加文件到暂存区
git add file(添加某个文件)
git add * (添加所有文件)

###提交文件
git commit -m”备注信息”

###查看历史
git log

###回复到上一次的提交
git reset –hard sha值

创建分支

git branch 分支名称

###切换分支
git checkout 分支名称

###查看分支
git branch

###合并分支
git merge 分支名称

###删除分支
git branch -d 分支名称

##git共享仓库

###Git 要求共享仓库是一个以.git结尾的目录。
mkdir test.git 创建以.git 结尾目录
cd test.git 进入这个目录
git init –bare 初始化一个空的共享仓库
git clone –bare 仓库地址 （将已有的仓库导出作为共享仓库）

向共享仓库push 或者pull 内容

git push 仓库地址 （提交内容到共享仓库）
git clone 仓库地址 分支名称 （克隆共享仓库内容）
git pull 仓库地址 分支名称 (下载或更新)

##git 一般使用步骤

第一步：先提交到本地
git add *
git commit -m”标记信息”
第二步：提交到共享仓库
git push 共享仓库地址 分支名称
第三步：更新最新内容
git pull 共享仓库地址 分支名称

可能产生的冲突

两个人同时修改了同一个文件的同一部分内容，并且，两人同时在本地做了提交操作，其中一个人先提交到共享仓库，另外一个人提交时，就会被要求先更新 git pull 在pull 的过程中会产生冲突。

##解决冲突

手动找到提示冲突的文件(里面内容大概长这样)

<<<<<<HEAD
你写的内容
==========
别人写的内容
>>>>>>>>32hri3

解决冲突 

手动把 <<<<<<HEAD    ==========   >>>>>>>>32hri3
这些东西删除，再保留自己想要的代码，之后需要再提交到本地再push到共享仓库当中
