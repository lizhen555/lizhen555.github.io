git 相关

发表于 2016-04-24 | 分类于 git

常用的bash命令

pwd： 查看当前目录  
cd：  切换目录    
ls：  查看当前目录下的内容     
mkdir： 创建目录（文件夹）                   
touch： 创建文件 如：touch index.html                        
cat： 查看文件全部内容  eg：cat index.html         
more/less ： 查看文件         
rm ： 删除文件 eg： rm -rf index.html            
rmdir ： 删除文件夹，只能删除空文件夹   
mv ： 移动文件或重命名 eg:mv index.html ./demo/index.html  
cp ： 复制文件

tab (自动补全，连按两次会将所有匹配的内容显示出来)
history (查看操作历史)
ssh (远程登陆，如：ssh root@gitlab.test.com)
> 和 >> 重定向 如：echo hell > READMR.md

3.底行模式： ：wq （保存并退出） e!(撤销更改，返回上一次保存状态)

Git工作原理

对于任何一个文件，在git内都有三种状态  
    1.已提交（committed）:表示该文件已经被安全地保存在本地数据库中  
    2.已修改（modified）：表示修改了某个文件，但还没有提交保存  
    3.已暂存（staged）：表示把已经修改的文件放在下次提交时要保存的清单中

Git管理项目时，文件流分为三个工作区域：
1.Git 的工作目录
2.暂存区域
3.本地仓库

git命令

配置用户
git config --global user.name "your name"  
git config --global user.email "your email"  
初始化仓库  
git init  
查看文件状态  
git status  
添加文件到暂存区  
git add file(添加某个文件)  
git add * (添加所有文件)   
提交文件
git commit -m"备注信息"  
查看历史  
git log  
回复到上一次的提交  
git reset --hard sha值
创建分支  
git branch 分支名称  
切换分支  
git checkout 分支名称  
查看分支  
git branch  
合并分支  
git merge 分支名称  
删除分支  
git branch -d 分支名称

git共享仓库

Git 要求共享仓库是一个以.git结尾的目录。

mkdir test.git 创建以.git 结尾目录
cd test.git 进入这个目录
git init –bare 初始化一个空的共享仓库
git clone –bare 仓库地址 （将已有的仓库导出作为共享仓库）

向共享仓库push 或者pull 内容

git push 仓库地址 （提交内容到共享仓库）
git clone 仓库地址 分支名称 （克隆共享仓库内容）
git pull 仓库地址 分支名称 (下载或更新)

git 一般使用步骤

第一步：先提交到本地
git add *
git commit -m”标记信息”
第二步：提交到共享仓库
git push 共享仓库地址 分支名称
第三步：更新最新内容
git pull 共享仓库地址 分支名称

可能产生的冲突

两个人同时修改了同一个文件的同一部分内容，并且，两人同时在本地做了提交操作，其中一个人先提交到共享仓库，另外一个人提交时，就会被要求先更新 git pull 在pull 的过程中会产生冲突。

解决冲突

手动找到提示冲突的文件(里面内容大概长这样)

<<<<<<HEAD
你写的内容
==========
别人写的内容
>>>>>>>>32hri3

解决冲突 

手动把 <<<<<<HEAD    ==========   >>>>>>>>32hri3
这些东西删除，再保留自己想要的代码，之后需要再提交到本地再push到共享仓库当中

版本回滚

git reset --hard 傻值 （工作区和版本记录都会变化）
git reset --soft 傻值 （工作区没有变化，版本记录变化，实际上是回退到暂存取）
git reset 傻值  （默认，回退到已修改状态） --mixed
git checkout -- 文件名  (从暂存区回退指定文件)

多人协作时,如何保证有一份安全的代码(里面保证没有任何bug)

创建平衡分支(develope 分支),Master分支让他受保护,想要往master分支上提交代码,必须要有权限.

忽略文件

在.gitignore 文件中指定哪些文件不去添加到仓库当中
使用 vi .gitignore 创建.gitignore文件,在vi命令行环境下添加需要忽略的文件

创建.gitignore文件：  vi .gitignore
指定不用添加的文件：
文件名/* (该文件里所有的文件都不用上传)
文件名/*.png (该文件中以.png结尾的文件都不上传)

开发者使用版本控制

1.把项目clone到本地后创建一个分支
2.所有代码都在自己的分支上做
3.提交到远程仓库
4.在远程仓库中创建一个合并请求发送给自己（合并到开发者分支）
5.合并后再在开发者分支发送合并请求到master分支

在pull的时候出项的错误

出项的错误：
fatal: Not a git repository (or any of the parent directories): .git
解决方法：
先通过git init 初始化然后再进行pull操作即可

在push的时候出现的错误

在使用 git push origin master 是出现以下错误：
fatal: 'origin' does not appear to be a git repository
fatal: Could not read from remote repository.

解决方法：（通过以下方法进行push操作）
$ git remote add origin git@github.com:defnngj/hello-world.git     //连接远程github项目
$ git push -u origin master     //将本地项目更新到github项目上去
