AngularJS

发表于 2016-07-22 | 分类于 AngularJS

使用AngularJS四个基本步骤：

1.创建模块  var app = angular.module('模块名'，[]);
2.创建控制器 app.controller('控制器名'，['$scope',function(){    。。。实现事件属性和方法    }])
3.绑定模块  ： ng-app = "模块名称"
4.绑定控制器 ： ng-controller = "控制器名称"

模块化

AngularJS构建应用（App）时是以模块化（Module）的方式组织的，即将整个应用划分成若干模块，每个模块都有各自的职责，最终组合成一个整体。
采用模块化的组织方式，可以最大程度的实现代码的复用

定义模块

1.创建模块
    AngularJS 提供了一个全局对象angular,在此全局对象下存在若干方法，其中angular.module()方法用来定义一个模块
        第一个参数：模块名称
        第二个参数：依赖模块名称
    var app = angular.module('app',[]);
2.创建控制器
    app.controller('控制器名称',['$scope',function($scope){
        $scope.test1 = 'xxx';
        $scope.students = [
            {name: 'zs',age: 33},
            {name: 'ls',age: 22}
        ]
    }])
3.绑定模块 ng-app = "模块名称"
    ng-app = "app"
4.绑定控制器
    ng-controller = "控制器名称"

数据绑定

数据单向绑定：
    1. {{属性}} : 插值语法（会出现闪烁）
         出现闪烁原因：浏览器先执行dom加载，再执行angular解释
        解决方法：
            添加 ng-cloak
            设置style--> [ng-cloak]{display:none}
        即在dom加载的时候把标签隐藏

    2. ng-bind="属性"
    3. ng-bind-template="{{属性}}"
数据双向绑定：
    双向绑定必须要借助表单元素 eg:
    <input type="text" ng-model="name">
    <p>{{name}}</p>

指令

HTML在构建应用（App）时存在诸多不足之处，AngularJS通过扩展一系列的HTML属性或标签来
弥补这些缺陷，所谓指令就是AngularJS自定义的HTML属性或标签，这些指令都是以ng-做为前缀  
例如ng-app、ng-controller、ng-repeat等。

内置指令

ng-app 指定应用根元素，至少有一个元素指定了此属性。
ng-controller 指定控制器
ng-show控制元素是否显示，true显示、false不显示（本质上是把css样式设置为none）    
ng-hide控制元素是否隐藏，true隐藏、false不隐藏
ng-if控制元素是否“存在”，true存在、false不存在
ng-src增强图片路径
ng-href增强地址
ng-class控制类名     （ng-class="{key:value}" value是bool类型）
ng-include引入模板
ng-disabled表单禁用
ng-readonly表单只读
ng-checked单/复选框表单选中
ng-selected下拉框表单选中
ng-click:  方法名（）
ng-repeat: 遍历  eg: ng-repeat:"stu in students"

自定义指令

AngularJS允许根据实际业务需要自定义指令，通过angular全局对象下的directive方法实现。

自定义指令格式： 第一个参数：指令名称  第二个参数：回调函数
    app.directive('demo',[function(){
        return{
            //E:元素 A：属性 C：当作类 M：可以以注释形式出现
            restrict:'EA',
            //template: '<div><p>dddd</p><span>ssss</span></div>',
            templateUrl: 'demo.html',   //外链模板
            replace: true  //是否替换原标签
        }
    }])

过滤器

作用：用来格式化数据
管道符：上次的结果当作下一次的参数进行传递

内置9大过滤器

1、currency:将数值格式化为货币格式
    <p>{{price | currency:'$'}}</p>
2、date:日期格式化，年（y）、月（M）、日（d）、星期（EEEE/EEE）、时（H/h）、分（m）、秒（s）、毫秒（.sss），也可以组合到一起使用。  
    <p>{{nowDate | date:'yyyy/MM/dd h:m:s'}}</p>
3、filter:在给定数组中选择满足条件的一个子集，并返回一个新数组，其条件可以是一个字符串、对象、函数  
    <p>{{course | filter: 'a'}}</p>
4、json:将Javascrip对象转成JSON字符串。
    <p>{{student | json}}</p>
5、limitTo:取出字符串或数组的前（正数）几位或后（负数）几位
    <p>{{student | limitTo: 2}}</p>
6、lowercase:将文本转换成小写格式
    <p>{{str | lowercase}}</p>
7、uppercase:将文本转换成大写格式
    <p>{{str | uppercase}}</p>
8、number:数字格式化，可控制小位位数
    <p>{{num | number:2}}</p>
9、orderBy:对数组进行排序，第2个参数可控制方向
    //true 是降序  false是升序
    <p>{{person | orderBy:'age':false}}</p>

自定义过滤器

第一个参数：过滤器的名称  
第二个参数：回调函数
返回值是一个函数，input 为自动传入的数据，是管道符前面的内容
app.filter('filterName',[function(){
    return function (input){
        console.log('Hi'+input);
    }
}])

##服务

内置服务

$location服务

//获取的是锚点后面的地址
$scope.url = $location.url();
//端口号
$scope.port = $location.port();
//主机名
$scope.host = $location.host();
//获取的锚点，从第二个锚点开始的值
$scope.hash = $location.hash();
//参数
$scope.search = $location.search();

$timeout服务

进行依赖注入后再使用
$timeout（function(){
    .......
},1000）;

$interval服务

$interval(function(){
    $scope.dateTime = new Date();
},1000);

停止定时器
$interval.cancel(定时器名称);
$timeout.cancel(定时器名称);

自定义服务

注意点：AngularJS 1.6以上版本网络请求不再使用 success error;而是使用 then catch 替代

$http服务-get

$http({
        url:"",
        method:"get",
        params:{key:value}, //参数
}).then(function(res){   //请求成功时，回调
        ..... 
}).catch(function(error){    //请求失败时，回调
        .....
});
}]);

$http服务-post

$http({
    url:"",
    method:"post",
    //post必须要设置请求头
    headers:{'Content-Type':'application/x-www-form-urlencoded'},            //请求头
    data:'name=xxx'            //请求参数（默认以json形式传递）
}).then(function(res){    //请求成功时，回调
    .....
}).catch(function(error){    //请求失败时，回调
    .....
});

配置块与运行块

配置块

配置块会在提供者注册和配置的过程中，对模块或者服务进行配置

通过config方法实现对模块的配置,AngularJS中的服务多数都对应一个provider,  
    用来执行与对应服务相同的功能或对其配置,比如$log、$http、$location都是内置服务，  
    相对应的“provider”分别是$logProvider、$httpProvider、$locationPorvider。 

    eg:

    var app = angular.module('app',[]);
    app.controller('testController',['$scope','$log',function($scope,$log){
        $log.debug('调试信息')；
        $log.debug('调试信息')；
        $scope.price = "hello";
    }]);

    //provider 在每一个服务当中都会有一个provider来对指定的功能进行配置
    app.config(['$logProvider',function ($logProvider){
        $logProvider.debugEnabled(false);
    }]);

    app.config(['$filterProvider',function ($filterProvider){
        $filterProvider.register('firstUpp',function(){
                //可以在配置过程当中，给过滤器添加过滤功能return function (input){
                    return input[0].toUpperCase + input.slice(1);    
                }    
        })
    }]);

运行块

服务也是模块形式存在的对且对外提供特定功能，前面学习中都是将服务做为依赖注入进去的,然后再进行调用，除了这种方式外我们也可以直接运行相应的服务模块，AngularJS提供了run方法来实现。 

run方法还是最先执行的，利用这个特点我们可以将一些需要优先执行的功能通过run方法来运行，比如验证用户是否登录，未登录则不允许进行任何其它操作。 
eg:
var app = angular.moudle("app",[]);

//ng 程序一执行就会调用run
app.run(['$http','$rootScope',function($http,$rootScope){
    $http({
        url: 'jsonp.php',
        method:'get',
        params:{ callback: 'xxx'}
    }).then(function (res){
        console.log(res);
        $rootScope.name = res;
    });
}]);

app.controller('testController',['$scope','$http',function($scope,$http){
    $scope.name = "testController";
}]);

##路由
在angular中路由就是锚点，用来做单页面应用程序的切换
锚点的变化又称为路由的变化
路由在angular中属于单独的一个模块

angular实现路由的步骤

1.引入angular—route.js
2.实例化模块时，注入ngRoute模块
    var app = angular.module('app',['ngRoute']);
3.配置路由模块
    //配置路由，监听哪些锚点变化
    app.config(['$routeProvider',function($routeProvider){
    //当锚点为指定值时，处理相应的逻辑;路由规定，在锚点前加上“/”
    $routerProvider.when('/index',{template:"<h1>首页</h1>", controller:"控制器名称"})
    .when('/about',{template:"<h1>关于</h1>"，, controller:"控制器名称"}})
    .otherwise({redirectTo:'/index'})   }]);
4.布局模块
    <div class="nav">
        <ul>
            <li><a href="#/index">首页</li>        
            <li><a href="#/about">关于</li>
        </ul>
    </div>
    <div class="content" ng-view>..显示相关页面的区域..</div>

注意在使用路由外链模板时
需要把template改为 templateUrl:”xxx.html”

路由传参

$routerProvider.when('/index/:page',{template:"<h1>首页</h1>", controller:"控制器名称"})

<li><a href="#/index/2">首页</li>
